OC中分类的使用

发表于 2015-05-05   |   分类于 iOS   |     |   阅读次数

现在有这么一个需求，如果对一个类在不更改代码的基础上要要为它扩充额外的方法，对于面向对象语言我们首先想到的就是面向对象的特性之一的继承。但是在ios中有一个更好用的语法，那就是分类。

注：以前的学习笔记。

分类是什么

现在有这么一个需求，如果对一个类在不更改代码的基础上要要为它扩充额外的方法，对于面向对象语言我们首先想到的就是面向对象的特性之一的继承。但是在ios中有一个更好用的语法，那就是分类（Catgory）。

为什么不用继承

  • 继承耦合度高。
  • 分类是给原类（继承中的父类）添加更多的方法，并且通过原 类来调用，没有产生新的类。

分类有什么优势

  • 如果把基类看作是一部手机，现在需要给这部手机增加一些功能，分类可以直接给手机增加功能，这些功能直接就可以使用；而继承就需要再造一部新的手机，然后才能为它添加功能。

分类和继承的区别？

  • 分类不能实例化，它只是给基类扩展了一些方法而已。
  • 通过继承得到的类能够实例化，是一个全新的类。
  • 分类是实例化基类来调用分类的方法，因为扩展的方法就是父类的方法，而继承需要实例化子类才能调用子类的方法。

一些分类的结论：

  • 在分类实现方法中可以访问原类中的成员变量，但是不能访问原类中的属性@property。
  • 分类中如果写了和原来中一样的方法，那么会调用方法会调用分类中的方法实现，最好不要这样做。
  • 在分类只能增加方法,不能增加成员变量。
  • 在分类中可以重新实现原类中的方法，但会将原类中的方法覆盖而失效。因为在执行对象成员方法的时候会优先去分类中查找，然后再去原类中去查找，最后去原类中去查找。但是一般不建议么做！
  • 如果一个类有多个分类，而且分类中有同名的方法那么最后编译的分类会将前面编译的分类覆盖并且执行后面编译的方法。

分类的应用：

  • 给常用的类增加一些经常用的方法,如框架里面的类。
  • 向对象添加非正式协议。
  • 有时候在处理类定义时,可能想要为其添加一些新方法。
  • 假如你参与了一个大型程序设计项目,并且作为项目的一部分,正在定义一个新类,新类中包含许多方法.你的任务是:为该类编写处理文件系统的方法.其他项目成员的任务负责以下方法:创建和初始化该类实例,对该类中的对象执行操作以及在屏幕上绘制该类对象的表示。
  • 假如你已经知道如何使库中的类(例如fundation的数组类,NSArray),并且意识到你希望该类实现了一个或者多个方法.当然,你可以编写NSArray类的新子类并实现新方法。

总结

  • 分类在实际运用中有很大的作用，因此在项目实战的时候要好好的体会！
